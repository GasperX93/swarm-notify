Encrypted notifications and messaging on Swarm — no servers, no cloud, no accounts.

1

Publish Identity

Publishes Alice's public keys and overlay to a Swarm feed at topic keccak256("swarm-identity-" + ethAddress). The data is plaintext — public by design. Anyone who knows Alice's ETH address can read this feed and get her encryption keys. Think of publishing your ETH address like sharing an email address.
3

Discover Bob

Alice looks up Bob's ETH address on Swarm to find his public key. In this demo, both addresses are visible on screen. In production, you'd share your ETH address the same way you share an email — paste it in a chat, put it on your website, or scan a QR code. Alice needs Bob's public key for ECDH key exchange.
4

Send Message to Bob

ECDH derives a shared secret from Alice's private key + Bob's public key. AES-256-GCM encrypts the message with this secret. The encrypted blob is uploaded to Swarm and Alice's mailbox feed (Alice→Bob) is updated. Only Alice and Bob can derive this key — neither party ever sends it.
6

Notify Bob (On-Chain)

ECIES-encrypts {sender, overlay, feedTopic} with Bob's public key and emits a Notification event on Gnosis Chain. Only needed for first contact — subsequent messages are found via feeds. Cost: ~22,000 gas (~0.00002 xDAI).

Note: This requires the sender's wallet to have xDAI on Gnosis Chain. The demo generates fresh keys with no funds, so this step will fail unless you fund Alice's address.
8

Check Inbox

Reads Bob's mailbox feed (Bob→Alice) from Swarm, derives the shared secret, and decrypts. This shows the reply from step 7.

Bob

ETH: —
PubKey: —
2

Publish Identity

Same operation for Bob. Both identities are now discoverable by ETH address on Swarm. Anyone can read the feed — privacy comes from encryption, not access control.
3

Discover Alice

Same in reverse — Bob looks up Alice's ETH address to get her public key. Both now have each other's keys — ready for encrypted communication. No key exchange server needed.
5

Read Messages from Alice

Reads Alice's mailbox feed (Alice→Bob) from Swarm. Derives the same shared secret using Bob's private key + Alice's public key — ECDH is symmetric, both sides compute the same key. Decrypts with AES-256-GCM. Neither party ever sent the key!
6

Poll Notifications

Queries Gnosis Chain for Notification events matching keccak256(bobAddress). ECIES-decrypts each with Bob's private key. Failed decryptions (spam or not for you) are silently discarded. This is how you discover new contacts who messaged you — no server needed.

Polling is free (read-only eth_getLogs). Only sending notifications costs gas.
7

Reply to Alice

Bob sends an encrypted reply using the same ECDH + AES-256-GCM flow, but on Bob's own mailbox feed (Bob→Alice). Two feeds per conversation — each owned by the sender. Merge both feeds by timestamp to see the full thread.
